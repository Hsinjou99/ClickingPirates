# ClickingPirates

An idle clicker game made in Unity by three people. This project was made in one day for CrimsonCode Hackathon 2026.
The final product is a Unity project containing the game since we haven't figured out how to export it as an executable. 
But all code, files, and assets are here.

## Reflection

### Activity description

As our first project using Unity, none of us expected anything big or impressive. We only hoped to have a simple game that works and does what we want it to. We wanted to make an idle clicker similar to Cookie Clicker with similar clicking mechanics and upgrades. 

### Technical decisions

The big decisions made were:
1. What game engine to use
2. What the core mechanics would be
3. What the structure of the program would look like

After some basic research into possible game engines like Unity, Godot, Phaser, etc., we stuck with Unity because of its popularity and abundance of guides online. Learning Unity would also be useful for any future indie projects involvolving game development.

The core mechanics and ideas for the game kept changing throughout the first few hours into the event since we still weren't sure of what we could do given our lack of knowledge and time constraint. We settled for a very simple idle clicker that has the player complete three stages by clicking an image and buying upgrades to reach a certain goal.

When we first started developing, we had no idea how to create a game in Unity. We followed video guides to get an idea of what the structure should look like and built our game off of that. Our final product was made by putting a bunch of images and buttons on the screen, and having events occur when those UI elements were activated.

### Contributions

Two of us worked on the UI and coding elements of the project, so we had to work on different parts of the game while trying not to negatively impact each other. Often times we ran into moments when one person had to wait for the other to finish first, which meant we weren't very effective with out time as one person would essentially be doing nothing for a while. 

The last member worked on the audio and visuals of our game. We couldn't find many free assets we wanted, so we had to create our own to fit what we wanted. This person created the scenes, sprites, and animations to match our theme and artstyle. They were in charge of making our UI look good and connecting the audio and animations to certain moments in the game.

### Quality assessment

We are all satisfied with how the project turned out. We started with no knowledge on game development, no direction, and no clear ideas. And we ended with a simple but working game that we are all proud of. When we first started, we wasted a lot of time stumbling around with no clear goals. So if we could redo this event, we would have decided on our approach and ideas first before diving into the development process.
